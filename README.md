🎬 Screenmatch (Sem Web)
##
Projeto desenvolvido em Java com Spring Boot para praticar consumo de APIs, manipulação de dados com Streams e uso de programação funcional (lambdas).
##
##
🚀 Objetivo do projeto:
##
O objetivo deste projeto é simular uma aplicação backend que:

- Consome dados de uma API externa

- Trabalha com séries, filmes e episódios

- Filtra e manipula dados utilizando Streams e Lambdas

- Aplica conceitos de Spring Boot em uma aplicação console
##
##
🛠️ Tecnologias utilizadas:
##
- Java

- Spring Boot

- Programação funcional (Lambdas)

- Streams API

- Jackson Databind

- Maven

- Consumo de API REST de filmes/séries
##
📁 Estrutura do projeto:
##
src/

 ├── main/java/br/com/alura/screenmatch
 
 │    ├── model/          # Classes de domínio (Serie, Episodio, etc)
 
 │    ├── service/        # Consumo de API e regras de negócio
 
 │    ├── principal/      # Menu e execução do sistema
 
 │    └── ScreenmatchApplication.java
 ##
 ##
⚙️ Como executar o projeto:
##
1. Clonar o repositório
git clone https://github.com/FlytrZz/screenmatch-sem-web.git

2. Abrir no Eclipse ou IntelliJ,
importar como Java Project e aguardar o download das dependências

3. Executar a aplicação
e rodar a classe principal:
ScreenmatchApplication.java

##
📊 Funcionalidades:
- Busca de séries e filmes via API
- Filtragem de dados com Streams
- Manipulação de listas de temporadas e episódios
- Ordenação e transformação de dados
- Menu interativo via console
